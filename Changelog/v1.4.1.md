# Phase 25: Domain URL Entry Toggle

**Date:** 2026-03-05

## Summary
Added a "Perform URL Entry" boolean toggle to the Kickoff Project form to control whether the `domain_url` column on the Vendor page is read/write or auto-parsed locally.

## Changes
### `projects` table (Database)
- Added `url_entry_enabled boolean DEFAULT false` to control this behavior globally per-project.

### `src/app/admin/new-project/page.jsx` & `actions.js`
- Added the "Perform URL Entry" UI toggle under Core Settings (default: off).
- Wired `url_entry_enabled` through Zod schema validation and into the DB insert.

### `src/app/vendor/[vendor_name]/[hash]/VendorForm.jsx` & `page.jsx`
- Re-ordered the Glide Data Grid to make `Domain URL` the very first column.
- Added a live "URL Entry" toggle button to the grid header allowing realtime modification.
- Added automatic parsing logic `parseDomainUrl()` to securely extract `https://hostname/`.
- If URL Entry Mode is OFF: `Domain URL` is strictly read-only and automatically parsed from the `published_url` input.
- If URL Entry Mode is ON: `Domain URL` becomes editable. Explicit manual entry or pasting into this column actively uses `parseDomainUrl()` to instantly format it into the expected footprint structure.
