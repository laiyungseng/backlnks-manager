# Changelog - v1.3.0

## [2026-03-03]

### Added
- **Admin Authentication System**: Built a real database-backed login loop against a new `admin_users` table with standard credentials. Tracks `last_login` ISO timestamp on every successful authentication.
- **Middleware Route Guard**: Implemented `src/middleware.js` that intercepts all `/admin/*` requests and forces a redirect to `/login` if the `df_admin_session_active` session cookie is absent.
- **Schema Builder**: New admin page at `/admin/schema-builder` providing a visual table builder with dynamic column configuration, data type dropdowns, and primary key selection. Generates live Supabase-compatible SQL and executes it via the Supabase Management API.
- **Sign Out Flow**: Global Sign Out button on the admin layout clears the persistent session cookie and redirects the user to `/login`.

### Changed
- **Session Persistence Migration**: Login page Server Action now sets a persistent `df_admin_session_active` cookie upon successful auth, bridging the gap between client-side state and server-side middleware verification.
- **Server Action Cache Invalidation**: Wired `revalidatePath('/admin', 'layout')` across Finalize and Lock Server Actions so the Dashboard and Completed Placements pages refresh instantly without manual page reload.
- **Kickoff UX**: Page now auto-scrolls to the top and displays a success notification banner immediately after a project is created.

### Fixed
- **Vendor Portal Bulk Paste**: Fixed an issue where pasting 100+ rows into the Glide Data Grid swallowed array state updates. Replaced default paste handling with a custom `onPaste` 2D React state mapping function. Auto-save now correctly fires on large pastes.
- **Admin Dashboard Instant Deletion**: Deleting a project from the Dashboard now removes it from the UI immediately without requiring a manual page refresh.

### Security
- **Admin Route Protection**: All `/admin/*` routes are now server-enforced via Next.js middleware. Directly navigating to any admin URL without a valid session redirects to `/login`.
- **Repository Privacy**: Updated `.gitignore` to explicitly exclude `MEMORY.md`, preventing local development logs from being exposed in the public repository.
- **Sign Out No Longer Wipes Credentials**: Removed the `persistToEnvFile('', '')` call from `logoutAction`. `.env` credentials are now preserved across sessions — sign out only deletes the session cookie.

### Infrastructure
- **Schema Builder Credential Modal**: Execution modal guides users to input their Supabase Project Reference and Management API token if missing before running any SQL.
- **Repository Sync**: Full codebase committed and pushed to GitHub at end of session.
