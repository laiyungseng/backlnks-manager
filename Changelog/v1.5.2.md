# v1.5.2 — 2026-03-05

## Bugfixes

### 1. Dashboard Edit Mode — Save Clears All Fields
**File:** `src/app/admin/actions.js` → `updateDashboardProjects`

**Root cause:** The action was reading edited values directly from the top-level project object (`p.project_name`, `p.vendor_name`, etc.), but `handleFieldChange` in `DashboardClient.jsx` stores all edits inside `p.project_details[0]`. Top-level fields don't exist, so every save wrote `undefined` to the DB, clearing all project metadata.

**Fix:** Read edited values from `p.project_details?.[0]` before merging into the DB record:
```js
const edited = p.project_details?.[0] || {};
details[0] = { ...details[0], project_name: edited.project_name, ... };
```

### 2. VendorLinkCopy — Copy Icon Removed by Mistake
**File:** `src/app/admin/placements/VendorLinkCopy.jsx`

**Root cause:** Previous session converted the copy button to an open-in-new-tab link and removed the copy icon, losing that functionality.

**Fix:** Restored both:

- Icon `<button>` → copies URL to clipboard with `Check` tick feedback
