## Phase 26 â€” Dashboard Optimizations & UI Bugfixes

### Problems Fixed
1. Edited parameters didn't display on screen after clicking "Save" without a manual browser reload.
2. Clicking "Approve" updated the database but the UI kept the old state (no green checkmark).
3. The Project Status badge retained the wrong unapproved label after approval until a page refresh.
4. Toggling into "Edit Mode" showed `dd/mm/yyyy` placeholders in Start Date and Deadline instead of the previously saved values.
5. Typo: approved-incomplete state label read "Inprocess" instead of "In Progress".

### Solutions
- **SSE Query:** Updated `src/app/api/realtime/dashboard/route.js` to include `price`, `price_type`, and `is_approved` in the polling payload every 4 seconds.
- **Optimistic UI â€” Save:** Added `setProjects([...editedProjects])` inside `handleSaveEdits()` in `DashboardClient.jsx` so the table reflects edits immediately on save.
- **Optimistic UI â€” Approve:** Added explicit `is_approved: true` mapping inside `handleApprove()` so the green checkmark appears instantly.
- **Input Null Routing:** Passed `value={project.vendor_name || ''}` (and equivalent for other fields) into `<input>` elements to prevent uncontrolled-to-controlled warnings and blank placeholders.
- **Date Formatter:** Split `start_date` and `deadline` with `.split('T')[0]` to produce `YYYY-MM-DD` strings that correctly bind to `<input type="date" />`.
- **Label Text:** Changed status fallback string to `details.status || 'In Progress'`.

## Phase 27 â€” Projects Table JSONB Reconstruction

### Summary
Migrated the `projects` table from a flat-column structure to a `project_details` JSONB array column, enabling easier schema evolution and nested field access.

### SQL Migration
**File:** `Changelog/phase27_projects_table_reconstruction.sql`

Key steps:
1. Added FK constraints (`projects_hub_project_id_fkey`, `placements_project_id_fkey`) idempotently.
2. Added top-level columns: `user`, `created_date`, `complete_date`, `project_details JSONB DEFAULT '[]'`.
3. Migrated all flat column data into `project_details[0]` via `jsonb_build_array(jsonb_build_object(...))`.
4. Converted `languages` array to `"languages-ratio": [{"lang-code": "...", "ratio": ...}]`.
5. Dropped 23 legacy flat columns (preserving `id`, `user`, `created_date`, `complete_date`, `project_details`).

## Bugfixes

### 1. Dashboard Edit Mode â€” Save Clears All Fields
**File:** `src/app/admin/actions.js` â†’ `updateDashboardProjects`

**Root cause:** The action read edited values from the top-level project object (`p.project_name` etc.), but `handleFieldChange` stores edits inside `p.project_details[0]`. Top-level fields don't exist â†’ every save wrote `undefined` to the DB.

**Fix:** Read from `p.project_details?.[0]` before patching:
```js
const edited = p.project_details?.[0] || {};
details[0] = { ...details[0], project_name: edited.project_name, ... };
```

### 2. VendorLinkCopy â€” Copy Icon Removed by Mistake
**File:** `src/app/admin/placements/VendorLinkCopy.jsx`

**Fix:** Restored both interactions:
- Main `<a>` button â†’ opens vendor page in new tab (`target="_blank"`)
- Icon `<button>` â†’ copies URL to clipboard with `Check` tick feedback

### 3. crypto.randomUUID Runtime Error
**Files:** `src/app/admin/new-project/page.jsx`, `src/app/admin/schema-builder/page.jsx`

**Root cause:** `crypto.randomUUID()` requires a secure context (HTTPS/localhost). Fails over plain HTTP on LAN IPs.

**Fix:** Replaced all call sites with a `genId()` fallback helper added after imports in each file.
