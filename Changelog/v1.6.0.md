# Changelog v1.6.0

## [1.6.0] - 2026-03-07

### Added
- Created an "Add New Project Info" group button on the Kickoff Project form to allow detailed nested configuration of SEO projects.
- Added dynamic Category and Sheet Name global string resolvers to "Dashboard" and "Placement" cards to support arrays of categories and sheet names seamlessly across the platform.
- Implemented the "Orientated" Design System for the Placements dashboard, featuring high-contrast technical typography, top gradient accents, and a modernized metadata grid.

### Changed
- Refactored `projectSchema.js` to ingest `project_info_json` as a nested Zod object array, grouping `sheet_name`, `category`, and nested `placement_target` lists.
- Removed flat `sheet_name` and `backlinks_category` from the new-project schema core fields, migrating them to the deeply nested JSON array representation.
- Upgraded the Kickoff Server Action (`/admin/new-project/actions.js`) to parse the nested lists, dynamically flatten the footprints into individual target arrays for the `projects_hub.targets` backend tracker, and write the nested array struct natively to `projects.project_details[0].project_info`.
- Standardized typography across the entire Admin Dashboard: all page titles (Dashboard, Vendors, Domains, etc.) now use `text-4xl font-bold` for platform-wide consistency.
- Performed minor codebase cleanup, moving temporary scripts to `_cleanup/`.

### Removed
- Removed direct inline editing support of `backlinks_category` on the Dashboard for grouped/nested projects to prevent complex nested array state overwrites on the simplified Dashboard grid UI.
