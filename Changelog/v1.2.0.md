# Changelog - v1.2.0

## [2026-03-01]

### Added
- **Backlink Category Enum Enforcement**: Changed "Kickoff Project" category input to a dropdown with strict `z.enum` validation (PBN, GP, Tier 2, etc.).
- **Sheet Name Tracking**: New `sheet_name` column in `projects` table to track source spreadsheets.
- **Auto-Timestamping**: `completed_at` ISO field added to `project_list` to log the exact moment a vendor fulfills their quota.
- **Database Finalization Awareness**: The Dashboard and Active Placements view now check for actual data presence in the `placements` table to verify finalization success.
- **Glide Data Grid Multi-Cell Deletion**: Integrated `onDelete` handler allowing vendors to highlight massive blocks of URLs/Remarks and press Delete/Backspace to clear them instantly, which automatically blank-targets their corresponding ISO timestamps and triggers a global Supabase Auto-Save.
- **Language Ratio Optimization**: Replaced single language field with a multi-language `languages` JSONB system allowing percentage distributions (totalling 100%) when generating staging targets via `phase21_languages.sql`.
- **Data Lock Security Workflow**: Finalized projects are gracefully locked (`is_locked=true`) from vendor modifications (`phase21_lock.sql`) mapping immediately into a new manual Admin Lock/Edit capability switch via `PlacementCard.jsx` Action Center.

### Changed
- **Dashboard Layout**: Reordered table columns to place "Category" before "Progress" for better flow.
- **KPI Metrics**: Refactored "In-flight" links to **"Pending Projects"**, which now correctly displays the number of incomplete projects instead of remaining URLs.
- **Default Expansion**: Set Active Placements cards to be **collapsed by default** to preserve dashboard space.
- **Status Badges**: Unified color logic for green "Completed" and "Completed & Finalized" statuses across all admin views.
- **Placement Labels**: Rebuilt Placement Card metadata string block into highly readable modular label clusters featuring specific tags like Language, Country, Category, and Qty.

### Fixed
- **Approve & Finalize Safeguard**: The button is now strictly disabled if the submitted URL count does not match the requested target quantity exactly.
- **Finalization Ghosting**: Solved issue where the "Finalize" button remained visible after a successful database upload by implementing a direct `placements` table check.
- **Backspace Deletion**: Added explicit Backspace key support to the Glide Data Grid vendor grid, so both Delete and Backspace keys trigger multi-cell clearing.
- **Ambiguous Dynamic Route Conflict Fix**: Destroyed legacy `/vendor/[hash]` route to fully prevent `next.js` compilation conflicts with the new `/vendor/[vendor_name]` format. Rebuilt edge compatibility tracking exclusively at `/vendor/[vendor_name]/page.jsx`.

### New Pages
- **Completed Placements**: New admin page (`/admin/completed`) showing only finalized projects that have been uploaded to the database. Active Placements now excludes these projects.
- **Settings Page**: New admin page (`/admin/settings`) for entering Supabase database credentials (Project URL & Anon Key) with placeholders, show/hide key toggle, and `.env` file persistence via API route.
- **Next-Gen Vendor Portal Pipeline**: Added explicit vendor dashboard tracking pages: `/vendor/[vendor_name]/inprogress/page.jsx` & `/vendor/[vendor_name]/completed/page.jsx`.
- **Branded Authentication Gateway**: Brand new `/page.tsx` home destination replacing plain redirect logic. Features glowing DF Logo animations tracking direct Auth endpoints `/login` and `/guest` validation interfaces.

### UI Enhancements
- **Collapsible Sidebar**: The admin sidebar can now be toggled between expanded (full labels) and collapsed (icons only) using a toggle button.
- **DF Branding**: Sign Out button icon replaced with a "DF" branded badge matching the header logo.
- **Vendor Portal Expansion Navigation**: The entire Vendor grid workspace has been retrofitted with a Collapsible React Sidebar explicitly tracking DF brand Identity strings over current projects, active projects, and past accomplishments dynamically.
