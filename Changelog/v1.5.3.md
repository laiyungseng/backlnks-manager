## Phase 28 — Manager Pages Workflow Enhancements

### Summary
Transitioned the Vendor and Domain Manager pages from an auto-save model to a manual "Edit Mode" with dirty-row tracking for optimized Suapbase writes. Corrected grid UI alignment issues.

### Frontend Changes
- **Manual Edit Mode**: Introduced `isEditMode` state. All cells read-only until toggled.
- **Trailing Row Entry**: Spreadsheet-style manual entry. Typing into the last row adds a new record and creates a new empty row.
- **Auto-Expanding Paste**: Grid now calculates required space on paste and dynamically adds empty records to fit the data.
- **Dirty Tracking**: Implemented a `Set` to track indices of modified rows. Only dirty rows are sent to `upsert` on "Save".
- **Auto-Save Removal**: Deleted the `useEffect` auto-save logic in both managers.
- **Grid Sync**: Added `key={isEditMode}` to `DataEditor` to ensure the canvas-based grid picks up state changes immediately.
- **Header Buttons**: Added "Edit Mode", "Save Changes", and "Cancel" buttons; removed redundant "Add" buttons.

## Bugfixes
1. **Cell Edit Offset**: Corrected the overlay positioning by ensuring the grid container has `relative` positioning.
2. **Crash on Edit**: Removed `provideEditor={true}` which was causing `TypeError: provideEditor is not a function`.
3. **Overwrite Prevention**: By filtering `vendors`/`domains` by `dirtyRows` before calling server actions, existing data is no longer touched unless explicitly edited.
