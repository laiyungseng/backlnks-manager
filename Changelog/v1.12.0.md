# Version 1.12.0

## [1.12.0] - 2026-03-20

### Added
- Hierarchical dashboard layout for `PlacementCard` components in Active and Completed views.
- Collapsed "Row" view showing Project ID, Vendor, Target URL, and Status.
- Expanded "Audit" view showing Project Name, Progress Bar, Metadata (Region, Dripfeed, Dates, Keyword), and per-target Distribution Table.
- Restoration of original "Dripfeed Status" terminology and metrics (`urls_per_day`, `dripfeed_period`).
- Integrated `VendorLinkCopy` into each individual project row for easier access.

### Changed
- Refactored `PlacementCard.jsx` to move high-density metadata into the expanded section.
- Renamed "Velocity Guard" back to "Dripfeed Status".
- Improved visual hierarchy with large typography and structured grids.

### Fixed
- Syntax error in grid container classes (`flex grid` corrected to `grid`).
- Redundant placement of progress bars and metadata blocks in `PlacementCard`.

### Phase 37 Updates (Same Session)
- **Breaking Change**: Removed `PlacementCard.jsx` and aggregated placements via `PlacementGroupCard.jsx` framework.
- Grouped `active` and `completed` dashboard views natively by Target URL & Vendor.
- Developed `PlacementProjectRow` for 12-column grid mapping of individual projects under each Target group.
- Restyled specific layout markers including status `...` dots, sub-table headers, category tracking chips, and inline action clusters (Lock / Verify / Finalize).

### Phase 38 Updates (Search & Global Metrics)
- **Added**: Real-time global Search Bar operating across Domains, Vendors, Projects, and Target Keywords.
- **Added**: High-level "Total Fulfillment" diagnostic widget to the main tracking headers.
- **Changed**: Adjusted expanded group containment background (`bg-slate-50/80`) to significantly enhance sub-project card legibility.
- **Refactored**: Migrated `CompletedPlacementsPage` aggregation logic to a client-boundary wrapper (`CompletedDashboardClient.jsx`) for seamless stateful browser rendering and search.

### Phase 39 Updates (Vendor-Centric Hierarchy)
- **Refactored**: Restructured Array Reducers natively merging projects by `Vendor Name` exclusively instead of splitting by Target URL variants.
- **Changed**: `PlacementGroupCard` Top Header typography and layout replaced. Replaced abstract target URL headers with strong Vendor Titles.
- **Changed**: Exchanged `Globe` target asset icon with `Users` vendor icon for clear object relations.
- **Changed**: Renamed "Recordings" summary metric to "Available Projects".
