# Changelog - v1.4.0

## [2026-03-04]

### Added
- **Vendor Manager**
  - Added new `/admin/vendor-manager` route.
  - Interactive grid allowing to create, read, update, and delete active vendors.
  - Includes dynamically populated column filters for `Vendor Name`, `Contact Email`, `Website`, `Status`, and `Remark`.
- **Domains Manager**
  - Added new `/admin/domains-manager` route.
  - Interactive grid allowing CRUD manipulation of domains.
  - Foreign-key mapping ensures users can edit the `Vendor` visually using the actual mapped `Vendor Name` parsed natively into dynamic text cells.
  - Comprehensive dynamically typed dropdown filters strictly tracking database schema: `Domain URL`, `Vendor`, `Domain Rating`, `Traffic`, `Domain Age`, `Spam Score`, and `Last Checked At`.

### Changed
- **Admin Navigation**
  - Added links for `Domains Manager` and `Vendor Manager` to the Sidebar component (`layout.jsx`).

### Security Updates
- **API Key Security (No NEXT_PUBLIC_ Exposure)**: 
  - `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` were deeply embedded in the browser JS bundle. Relabelled them to purely `SUPABASE_URL` and `SUPABASE_ANON_KEY` to strip them from client exposure.
  - Corrected mismatched JSON credential field names in `schema-builder/actions.js` to match DB-stored field names (`supabase_url` / `supabase_published_key`).
- **SSE Realtime Stream**: 
  - Removed client-side Supabase WebSocket from the Dashboard (eliminating the final vector for browser token exposure).
  - Built `src/app/api/realtime/dashboard/route.js` – an SSE route emitting named `projects` events securely polled from the server every 4 seconds.
- **Database Schema Consolidation**: Phase 24 Migration Finalized.
  - Dropped structural mapping dependencies by replacing legacy `project_list` and `project_targets` tables completely inside the DB pipeline.
  - Codebase swept of all invalid legacy table references.
  - `admin/actions.js` Delete flow cleaned to exclusively manage `placements` → `projects_hub` → `projects`.

### Fixed
- **Dashboard Silenced Project Deletion**: Fixed an issue where DB table foreign-key constraints on the legacy `project_targets` and `project_list` tables prevented `projects` deletion. Explicit sweeps added to gracefully unlink them if present, bypassing SQL exception failures on live platforms.
