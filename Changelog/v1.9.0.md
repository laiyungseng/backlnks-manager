# Drive-Future Backlink Manager - Changelog v1.9.0

## Phase 34: Project Display, Metadata & Vendor UX

**Date:** 2026-03-11

### Overview
Resolved issues with missing project metadata and display in the vendor portal and admin dashboard, and improved the sidebar toggle experience.

### Frontend Changes
- `src/app/vendor/[vendor_name]/inprogress/page.jsx`: Added `project_name` to the Supabase select query to ensure project names are displayed.
- `src/app/admin/placements/PlacementCard.jsx`: 
  - Added display for Dripfeed period (e.g., "(7 DAYS)").
  - Added visibility for project **Owner**, **Created Date**, and **Done Date** (Completed Date).
  - Fixed `ReferenceError` for `Clock` and `CheckCircle2` by adding missing Lucide-React imports.
- `src/app/vendor/[vendor_name]/layout.jsx`: 
  - Updated sidebar toggle to use `PanelLeftOpen` and `PanelLeftClose` icons.
  - Added `title` tooltips for Expand/Collapse states to improve identifyability.
- `src/app/admin/page.jsx` & `src/app/admin/completed/page.jsx`: Added dripfeed fields to queries to ensure UI hydration.

### Backend & API Changes
- `src/app/api/realtime/dashboard/route.js`: Added `dripfeed_enabled`, `dripfeed_period`, and `urls_per_day` to the SSE select query.
- `src/app/admin/new-project/actions.js`: Updated project creation to explicitly set `created_date` using `new Date().toISOString()`.
- `src/lib/placementProcessor.js`: Verified that `completed_date` is correctly updated during the project finalization flow.
