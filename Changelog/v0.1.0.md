# Drive-Future Changelog

All notable changes to the Drive-Future Backlink Manager project will be documented in this directory.

## [v0.1.0] - 2026-02-26 - Initial Setup & Workflow 1 (Admin Dashboard)

### Added
- Created the core Next.js architecture (App Router, Tailwind CSS, lucide-react).
- Added Global Admin Sidebar (`/src/app/admin/layout.jsx`) for navigation.
- Created Main Dashboard (`/src/app/admin/page.jsx`) which pulls and displays all active projects directly from the Supabase `projects` table.
- Added Project Kickoff Form (`/src/app/admin/new-project/page.jsx`) utilizing modern React 19 hooks (`useActionState`, `useFormStatus`).
- Built Server Action (`src/app/admin/new-project/actions.js`) logic to handle secure, multi-table generation:
  - Generates a UUID `vendor_token`.
  - Inserts the new project into the `projects` table.
  - Automatically loops and performs a bulk insert into `placements` table based on ordered `quantity`.
- Established strict Zod Input Schemas (`src/schemas/projectSchema.js`) adding auto-validation for `target_url` missing `https://`.
- **(Workflow 2):** Developed dynamic Vendor UI Sandbox (`/src/app/vendor/[vendor_token]/page.jsx`) restricting global navigation.
- **(Workflow 2):** Built interactive `VendorForm.jsx` rendering read-only tables alongside strict `published_url` input schemas.
- **(Workflow 2):** Authored Server Action (`/src/app/vendor/[vendor_token]/actions.js`) handling bulk array modifications.
- Initialized Supabase node client (`src/lib/supabase.js`) handling the frontend-to-database connection layer.
- Redirected the default Root Route (`/`) directly to the `/admin` Dashboard area.
- **(Dashboard Update):** Expanded Admin Dashboard to render Project IDs, Start Dates, Created At tracking, and Updated At tracking.
- **(Dashboard Update):** Added robust Delete Project functionality via Server Actions (`src/app/admin/actions.js`) linking native HTML forms directly to Supabase deletion schemas to maintain relational logic.
- **(Workflow 3):** Upgraded Admin Dashboard Server Action variables to natively render a visual progress bar (e.g. `1/2` Placements) mapping true data from the `projects` to `placements` tables.
- **(Workflow 3):** Wrote autonomous logic inside the vendor portal save action (`/vendor/[vendor_token]/actions.js`) that safely auto-completes parent `projects` if the URL array mathematical capacity reaches 100%.
- **(Workflow 4):** Created Parallax Vendor Tracker UI (`/admin/placements/page.jsx`) loading isolated nested tables mapping exact subproject (placement) IDs to their overarching project targets. Applied dynamic rendering of table variable headers strictly for empty state scenarios per user condition.
- **(Workflow 4):** Added `VendorLinkCopy.jsx` component inside Dashboard returning a 1-click clipboard navigator copy for extracting temporary hidden vendor URLs (`/vendor/[vendor_token]`) generated during kickoff.

### Fixed
- Fixed critical `500 Internal Server Error` caused by native Windows terminal copying `UTF-16LE` encoding into the `.env` file instead of `UTF-8`.
- Fixed the Vendor Portal routing param extraction to natively support both Next.js 14 objects and Next.js 15 asynchronous param promises to prevent false-positive extraction failures. 
- Diagnosed the Vendor Portal `Unauthorized` redirect bug via Browser Subagent and identified it as a strict PostgreSQL Error 42703: `column vendor_token does not exist` inside the `placements` table.
- Fixed the "Kickoff Project" button to use pure white, bold text for accessibility.
- Converted input fields default placeholder text color to absolute black (`text-gray-900`) for high visibility when typing.

### Security
- **Layer 1 Defense:** Implemented Cryptographic Secure Token matching off the URL parameter natively forcing dynamic Next.js routing.
- **Layer 2 Defense:** Added Anti-Snooping route traps (`/src/app/vendor/page.js` and `/src/app/unauthorized/page.jsx`) preventing base route traversal.
- **Layer 3 Defense:** The Database Bulk `UPDATE` explicitly attaches `.eq("vendor_token", token)` to all execution operations alongside individual `id` matching. This acts as a software replica of Supabase RLS, preventing ID enumeration.
- Placements placeholder ingestion logic requires active UUID `vendor_token` before provisioning rows.
- Data integrity verification is established fully server-side before database commits.
